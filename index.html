FP adaptation

  • favor val, minimize var
  • no loops
  • return sane values
  • Mixed project with existing Java - (Maven, Spring, Jpa)

Worked

  • Combinators in collection api (map, filter, flatMap, foreach, groupBy)
  • return something sensible, avoid Unit/void
  • Better types
  • match/case
  • higher order functions

Failed

  • Scala Enumeration, went back to Java enums
  • Hiding mutability of JPA, collections and objects
  • Too much types, generics and traits with JPA
